# Silicon Intern Tracker — Summer 2027

A single-page dashboard that surfaces **hardware design and verification**
internships for Summer 2027, split into two lanes:

- **Design** — RTL, ASIC, CPU microarchitecture, SoC, physical design, synthesis, static timing (digital only — analog/RF/mixed-signal excluded)
- **Verification** — DV, UVM, functional/formal verification, post-silicon, validation, DFT

It updates itself on a schedule with zero scraping on your part.

## How it works

```
 vanshb03/Summer2027-Internships          <- community list, scrapes career
   (listings.json, refreshed hourly)         pages across the industry hourly
              │
              ▼
 .github/scripts/aggregate.py             <- runs in GitHub Actions every 6h
   • keep active Summer 2027 roles
   • match your Design / Verification keywords
   • drop everything else (incl. software "validation" false positives)
              │
              ▼
        data.json  (committed by the Action)
              │
              ▼
        index.html  (GitHub Pages) renders the two lanes
```

You are **not** scraping Nvidia/Apple/AMD yourself — those big semis sit behind
Workday/Eightfold bot-protection and would be a maintenance sink. Instead you
piggyback on a community list that already watches hundreds of career pages, and
you filter it down to only what you care about.

**Honest limitation:** that community list skews software-heavy and can lag on
hardware roles, and big-semi postings mostly open **Sept–Nov**. Early in the
season the dashboard will be nearly empty — that's expected, not a bug. Treat it
as a fast first alert, not the entire hardware internship market. Keep applying
directly on company career pages too.

## Setup (about 5 minutes)

1. Create a new **public** repo on GitHub, e.g. `silicon-intern-tracker`.
2. Drop these files in at the same paths:
   ```
   index.html
   data.json
   README.md
   .github/scripts/aggregate.py
   .github/workflows/update.yml
   ```
3. Commit and push.
4. **Settings → Pages** → Source: *Deploy from a branch* → branch `main`, folder
   `/ (root)` → Save. Your site goes live at
   `https://<your-username>.github.io/silicon-intern-tracker/`.
5. **Settings → Actions → General → Workflow permissions** → set to
   *Read and write permissions* (so the Action can commit `data.json`). Save.
6. Go to the **Actions** tab → *Sync internship data* → **Run workflow** once to
   populate `data.json` immediately instead of waiting for the next cron.

That's it. From then on it refreshes every 6 hours on its own.

## Tuning it

Everything you'd want to change lives at the top of
`.github/scripts/aggregate.py`:

- **`SEASONS`** — currently `{"Summer"}`. Add `"Fall"`, `"Winter"`, `"Spring"`
  to also catch off-season 2027 roles (useful for earlier signal / co-ops).
- **`DESIGN_KEYWORDS` / `VERIFICATION_KEYWORDS`** — the phrases that route a role
  into each lane. Add anything specific you're targeting.
- **`STRONG_TOKENS` / `HARDWARE_GATE`** — the guard that keeps out non-hardware
  "validation"/"design engineer" noise. Loosen or tighten to taste.

Run it locally to preview the effect before pushing:

```bash
python3 .github/scripts/aggregate.py   # rewrites data.json
```

## Adding more sources (optional, later)

`SOURCES` in the aggregator is just a list of raw-JSON URLs with the community
list's shape. To pull specific startups directly (Etched, etc.), most modern
ATSs expose clean public JSON you can add a small fetcher for:

- **Greenhouse:** `https://boards-api.greenhouse.io/v1/boards/<token>/jobs`
- **Lever:** `https://api.lever.co/v0/postings/<token>?mode=json`
- **Ashby:** `https://api.ashbyhq.com/posting-api/job-board/<token>`

Normalize their fields to the same record shape
(`company_name`, `title`, `url`, `locations`, `season`, `sponsorship`,
`date_posted`, `active`, `is_visible`, `id`) and they'll flow through the same
filter. The big semis are the hard ones and deliberately left out.

## Files

| File | Purpose |
|---|---|
| `index.html` | The dashboard. Self-contained; fetches `./data.json`. |
| `data.json` | Filtered roles, regenerated by the Action. |
